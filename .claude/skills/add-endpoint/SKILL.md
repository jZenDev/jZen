# Adding a jZen backend endpoint

jZen is **proto-first and contract-first**. A resource method names only domain/proto types; the
framework negotiates the wire format via `X-Zen-Transport` (json/protobuf). Follow these steps in
order. Read `docs/architecture/BLUEPRINT.md` (transport seam) and `STANDARDS.md` ("OpenAPI and
the REST surface") first if
unsure.

## 1. Declare the payload proto

Add request/response messages to `proto/zen/v1/<domain>.proto` (one message per payload; there is
**no** generic envelope). Cross-cutting shapes (`ZenError`, `PageRequest`) already live in
`common.proto` — reuse them. Set `option java_package = "zen.proto.v1";` (bare `zen`, ADR-006).
Timestamps are epoch-ms `int64` with `_ms` suffix by convention.

## 2. Write the resource — return `Response`, not the bare proto

SmallRye cannot introspect protobuf classes (a bare-proto return type emits 130+ garbage builder
schemas). So the resource returns `jakarta.ws.rs.core.Response` and points at a schema by ref:

```java
@GET @Path("/{id}")
@Produces({MediaType.APPLICATION_JSON, "application/x-protobuf"})
@APIResponse(responseCode = ZenStatus.OK,   // ZenStatus is an extendable constant interface (ADR-003)
    content = {
      @Content(mediaType = MediaType.APPLICATION_JSON, schema = @Schema(ref = "MyMessage")),
      @Content(mediaType = "application/x-protobuf", schema = @Schema(ref = "MyMessage"))})
@APIResponse(responseCode = ZenStatus.NOT_FOUND, description = "... (ZenError)")
public Response get(@PathParam("id") String id) {
  return Response.ok(toProto(entity)).build();     // errors: Response.status(...).entity(zenError).build()
}
```

`server/zen-identity/.../user/AdminUserResource.java` and `.../auth/AuthResource.java` are the
reference resources. Use `ZenStatus` constants for `responseCode` (never raw `"200"`). Map the
Panache entity ⇄ proto with **MapStruct** (or a hand `toProto`, as AdminUserResource does).

## 3. Supply the component schema in the app's static OpenAPI

The clean model schema is **not** generated from the proto class — add it to the app's
`apps/<app>/<app>_server/src/main/resources/META-INF/openapi.yaml` under `components.schemas`, with
proto3 camelCase field names. SmallRye merges the annotation-scanned **paths** on top of this base.
(Long-term this file is generated by a protoc→OpenAPI step; today it's hand-authored.)

## 4. Framework vs app placement

- Reusable/identity resources (auth, user admin) belong in the **framework** library `zen-identity`
  so every app inherits them (ADR-001, ADR-005). Domain-specific resources belong in the app server.
- **Any library module contributing a resource/provider/bean MUST run `jandex-maven-plugin`** —
  without `META-INF/jandex.idx` Quarkus silently never discovers it. `zen-transport` is the reference.
- **Do not add `quarkus-rest-jackson` server-side.** It hijacks `application/json` and 500s on proto.
  The `Protobuf`/`ProtoJson` writers own serialization.

## 5. Regenerate and test

Run the `sync-contracts` skill (`task generate:proto generate:api`, then `task sync:contracts`).
Add a `@QuarkusTest` in the app server module asserting **both** transport modes and the `ZenError`
error path (see `AdminUserResourceTest`), then `task test:apps:server`.

## Admin/list endpoints (ADR-005)

The react-admin panel is JSON-only. List endpoints return a **bare JSON array + `Content-Range`**
header (`ra-data-simple-rest` convention), each element the declared proto rendered with
`JsonFormat.printer().alwaysPrintFieldsWithNoPresence()`. Add `Content-Range`/`Accept-Ranges` to the
app's CORS `exposed-headers`. See `AdminUserResource.list`.
